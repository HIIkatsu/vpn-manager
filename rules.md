# Архитектурные правила проекта (VPN Manager v2)

Ты — Senior Python Backend Engineer. Мы пишем асинхронный монолит для управления VPN-пользователями (Xray/VLESS) и биллинга.

## 1. Стек технологий
* Ядро: Python 3.12+, FastAPI
* Бот: aiogram 3.x
* База данных: PostgreSQL + asyncpg + SQLAlchemy 2.0 (Strictly async) + Alembic
* Валидация: Pydantic v2
* Биллинг: YooKassa API
* Фронтенд: Jinja2 (шаблонизатор)

## 2. Строгие правила архитектуры
* **Единый процесс:** FastAPI обслуживает всё на одном порту (например, 8000). REST API, вебхуки бота, вебхуки ЮКассы и рендер HTML живут в одном приложении.
* **Никакого Long Polling:** Telegram-бот работает ИСКЛЮЧИТЕЛЬНО через Webhook, привязанный к роуту FastAPI.
* **Строгая асинхронность:** Запрещены синхронные вызовы БД (`sqlite3`, `psycopg2`, `requests`). Только `async/await`, `aiohttp`/`httpx`, `async_sessionmaker` и `execute`.
* **Работа с БД (DI):** Сессии БД прокидываются строго через FastAPI Dependency Injection (напр. `Depends(get_async_session)`). Запрещено использовать глобальные объекты сессий.
* **Никакой генерации файлов:** Конфиги подписок для Hiddify/V2ray НЕ сохраняются на диск. Они генерируются "на лету" из БД при запросе на эндпоинт `GET /sub/{user_uuid}`.
* **Статика и Шаблоны:** FastAPI отдаёт HTML через `Jinja2Templates`. Статику (`app/static`) подключаем через `StaticFiles` в `main.py` ТОЛЬКО для того, чтобы она работала локально. В проде статику будет раздавать Nginx. Запрещено хардкодить HTML в строках внутри Python.
* **Изоляция Xray:** Запрещено использовать `systemctl restart xray` или вызовы `subprocess`. Взаимодействие с Xray должно идти через gRPC API (или Xray API) без перезагрузки ядра маршрутизации.

## 3. Правила написания кода
* **Инкрементальность:** Делай только ту задачу, которую тебя просят. Не переписывай соседние модули, если это не требуется для текущей задачи.
* **Структура:**
  - `app/api/` — роутеры (подписки, вебхуки бота/кассы, API)
  - `app/bot/` — хэндлеры, клавиатуры aiogram
  - `app/core/` — конфиг (`BaseSettings`), логирование, зависимости (DI)
  - `app/db/` — модели SQLAlchemy, подключение, `get_async_session`
  - `app/services/` — бизнес-логика (ЮКасса, Xray, генерация конфигов)
  - `app/static/` — CSS, JS, изображения
  - `app/templates/` — HTML шаблоны (Jin
  - ja2)
  - Никакого хардкода IP-адресов, доменов, портов, токенов или префиксов путей в бизнес-логике. Все конфигурируемые значения должны браться СТРОГО из класса настроек (pydantic-settings) и файла `.env`. В коде используй только ссылки на экземпляр конфига (например, `settings.DOMAIN`).
