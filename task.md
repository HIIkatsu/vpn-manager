# План разработки VPN Manager v2

## Этап 1: Базовый скелет и БД
- [ ] 1.1. Инициализировать проект: создать структуру папок `app/api`, `app/bot`, `app/db`, `app/core`, `app/services`.
- [ ] 1.2. Настроить `app/core/config.py` (через `pydantic-settings` для чтения `.env`).
- [ ] 1.3. Поднять асинхронное подключение к PostgreSQL в `app/db/database.py`.
- [ ] 1.4. Написать SQLAlchemy-модели: `User` (telegram_id, vless_uuid, is_active, sub_end_date) и `Payment` (payment_id, amount, status).
- [ ] 1.5. Настроить Alembic и создать первую миграцию.

## Этап 2: Ядро и Telegram-бот
- [ ] 2.1. Создать базовое приложение FastAPI в `main.py`.
- [ ] 2.2. Инициализировать aiogram 3 и настроить роут вебхука в FastAPI (`POST /webhook/telegram`).
- [ ] 2.3. Написать базовые хэндлеры бота: `/start`, проверка профиля, кнопка "Продлить подписку". Вынести их в `app/bot/handlers.py`.
- [ ] 2.4. Привязать хэндлеры к БД (регистрация нового юзера, генерация уникального vless_uuid).

## Этап 3: Биллинг (ЮKassa)
- [ ] 3.1. Реализовать сервис `app/services/yookassa_srv.py` для генерации ссылок на оплату.
- [ ] 3.2. Добавить эндпоинт `POST /api/webhook/yookassa` для приема статусов платежей.
- [ ] 3.3. Реализовать логику продления: при статусе "succeeded" обновлять `sub_end_date` юзера в БД и отправлять ему уведомление через экземпляр aiogram bot.

## Этап 4: Подписки (Генерация конфигов Hiddify)
- [ ] 4.1. Создать роут `GET /vpn-{secret_prefix}/{user_uuid}`.
- [ ] 4.2. Написать сервис генерации конфига: если юзер активен, формировать правильный JSON/Base64 строку конфигурации.
- [ ] 4.3. Настроить отдачу конфига с правильными HTTP-заголовками (profile-title, profile-update-interval) напрямую из FastAPI, без файлов.

## Этап 5: Интеграция с Xray
- [ ] 5.1. Написать интерфейс/сервис в `app/services/xray_srv.py` для добавления и удаления UUID в памяти Xray (через gRPC).
- [ ] 5.2. Интегрировать вызовы Xray в логику покупки и истечения подписки.
